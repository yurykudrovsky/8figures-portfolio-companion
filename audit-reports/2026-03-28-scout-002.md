#### W3 — `$any()` type bypass in chat template — **CONFIRMED**

- **File:** `frontend/src/app/features/chat/components/chat-page/chat-page.component.html:49`
- **Evidence:**

- **Rule violated:** "No implicit any — EVER" / template type-check bypass (CLAUDE.md — TypeScript Rules)
- **Recommendation:** Replace with `(ionInput)="onInputChange($event.detail.value)"` which is the typed Ionic `ionInput` event detail.

#### W4 — Non-null assertion on portfolio signal — **CONFIRMED**

- **File:** `frontend/src/app/features/chat/components/chat-page/chat-page.component.ts:112`
- **Evidence:**
  ```typescript
  if (!text || this.isStreaming() || !this.portfolio()) return;  // line 110
  const portfolio = this.portfolio()!;                           // line 112 — unjustified !
  ```
- **Rule violated:** TypeScript strict mode — non-null assertion suppresses compiler narrowing without a typed local variable.
- **Recommendation:** Assign to a typed const before the guard: `const portfolio = this.portfolio(); if (!portfolio) return;`

- **File:** `frontend/src/app/features/portfolio/components/portfolio-dashboard/portfolio-dashboard.component.ts:79–87`

#### NEW-1 — Template non-null assertions on `portfolio()` signal in portfolio dashboard — **NEW WARNING**

- **File:** `frontend/src/app/features/portfolio/components/portfolio-dashboard/portfolio-dashboard.component.html`
- **Lines affected:** 59, 62, 65, 67, 71, 73, 80 (seven occurrences)
- **Evidence (representative sample):**
  ```html
  <!-- line 59 -->
  <h1 class="total-value">{{ formatCurrency(portfolio()!.totalValue) }}</h1>
  <!-- line 62 -->
  [ngClass]="portfolio()!.dailyChange >= 0 ? 'gain' : 'loss'"
  <!-- line 80 -->
  @for (holding of portfolio()!.holdings; track holding.id) {
  ```
  All seven usages appear inside the `@if (!loading() && !error() && portfolio())` guard block (line 56), so the runtime is safe. However, Angular's strict template type checker still sees each `portfolio()` call as returning `Portfolio | null`, requiring the `!` assertion to compile. This is the same category of violation as W4 — the non-null assertion suppresses the type system rather than narrowing to a typed local.
- **Rule violated:** TypeScript strict mode; the template type checker correctly infers `Portfolio | null` and the `!` bypass is unjustified when a typed local binding is available.
- **Recommendation:** Use `@let p = portfolio()!;` (Angular 17.2+ `@let` syntax) or extract a typed component property inside the guard to eliminate all seven `!` assertions in a single change.
